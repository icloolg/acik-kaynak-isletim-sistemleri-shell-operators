01 Üç Standart Dosya

Bir komut çalıştırdığında Linux her programa otomatik olarak 3 "dosya kanalı" açar.

FD: 0
stdin
Standart girdi.
Varsayılan: Klavye
FD: 1
stdout
Standart çıktı.
Varsayılan: Terminal ekranı
FD: 2
stderr
Standart hata.
Varsayılan: Terminal ekranı
🎙 Günlük Hayat Örneği

Fabrika analojisi: stdin = hammadde girişi, stdout = ürün çıkışı, stderr = kalite kontrol alarmı.

02 Stdout Yönlendirme

Normalde ekrana gelen çıktıyı bir dosyaya yönlendiririz.

OperatörDavranışDikkat!
> dosya Dosyayı sıfırdan yazar Eski içerik silinir!
>> dosya Dosyanın sonuna ekler Güvenli, eski içerik korunur
Basit

Tarih bilgisini dosyaya kaydet

~1 dk
bash
$date > bugunun_tarihi.txt
$cat bugunun_tarihi.txt
Mon Apr 13 14:32:08 TRT 2026
Ekran yerine bugunun_tarihi.txt dosyasına yazıldı. Dosya yoksa oluşturulur, varsa sıfırlanır.
Orta

Listeye bilgi ekle, üstüne yazma

~2 dk
bash
$ls -la /etc > sistem_dosyalari.txt
$echo "--- Liste tamamlandı ---" >> sistem_dosyalari.txt
$cat sistem_dosyalari.txt
total 1234
drwxr-xr-x 132 root root 12288 Apr 13 14:00 .
...
--- Liste tamamlandı ---
İlk satır > ile yeni dosya oluşturdu. İkinci satır >> ile sonuna ekledi. Eğer > kullansaydık, ls çıktısı kaybolurdu.
Zor

Döngüde log dosyası oluştur

~3 dk
bash
$for i in 1 2 3 4 5; do
>   echo "[$(date +%H:%M:%S)] İşlem $i tamamlandı" >> islem_logu.txt
>done
$cat islem_logu.txt
[14:32:01] İşlem 1 tamamlandı
[14:32:01] İşlem 2 tamamlandı
[14:32:02] İşlem 3 tamamlandı
[14:32:02] İşlem 4 tamamlandı
[14:32:02] İşlem 5 tamamlandı
Her iterasyonda >> ile yeni satır ekleniyor. Zaman damgalı log dosyası oluştu. Apache, Nginx gibi web sunucuları tam bu şekilde çalışır.
🌐
Gerçek Hayat: Web SunucularıApache, Nginx, PostgreSQL — tüm servisler olayları bu yöntemle log dosyasına yazar. Sistem yöneticileri hataları bu loglardan bulur.

03 Stdin Yönlendirme

Programın klavyeden değil, bir dosyadan ya da doğrudan terminalden okumasını sağlarız.

OperatörDavranış
< dosyaDosyayı stdin olarak kullan
<< KELIMEHere-document: KELIME görene kadar oku
Basit

Dosyadan sıralama

~1 dk
bash
$echo -e "Zeynep\nAhmet\nBurak\nAyşe" > isimler.txt
$sort < isimler.txt
Ahmet
Ayşe
Burak
Zeynep
sort komutu normalde klavyeden bekler. < ile dosyayı stdin olarak verdik.
Orta

Stdin ve stdout birlikte yönlendir

~2 dk
bash
$sort < isimler.txt > sirali_isimler.txt
$cat sirali_isimler.txt
Ahmet
Ayşe
Burak
Zeynep
Tek satırda hem girdi hem çıktı yönlendirme! isimler.txt → sort → sirali_isimler.txt
Zor

Here-Document — Çok satırlı metin girişi

~3 dk
bash
$cat << BITIS
>Merhaba,
>Yedekleme tamamlandı.
>Tarih: $(date)
>BITIS
Merhaba,
Yedekleme tamamlandı.
Tarih: Mon Apr 13 14:32:08 TRT 2026
BITIS kelimesini görene kadar tüm satırlar stdin olarak alındı. Değişken ve komut ikamesi ($(date)) de çalıştı.
📧
Gerçek Hayat: Otomatik E-posta / Rapor ŞablonlarıCron ile çalışan script'ler, here-document kullanarak otomatik e-posta veya rapor şablonu oluşturur. Ayrıca bash script içinde SQL sorgusu göndermek için de kullanılır.

04 Stderr Yönlendirme

Hata mesajları stdout'tan ayrı bir kanaldadır — bu yüzden ayrıca yönlendirilmesi gerekir.

🎙 Anlatım Notu

stdout ve stderr ikisi de ekranda görünür ama farklı kanallardır!

OperatörDavranış
2> dosyaStderr'i dosyaya yaz (üzerine)
2>> dosyaStderr'i dosyaya ekle
2>&1Stderr'i stdout ile aynı yere gönder
2>/dev/nullHataları tamamen yok say
Basit

Hata mesajını dosyaya yakala

~1 dk
bash
$cat olmayan_dosya.txt 2> hata_logu.txt
 # Ekranda hiçbir şey görünmez
$cat hata_logu.txt
cat: olmayan_dosya.txt: No such file or directory
Hata ekrana çıkmadı, hata_logu.txt dosyasına gitti. Burası çoğunlukla gözden kaçan ama kritik bir kavram.
Orta

Başarılı çıktı ve hataları ayır

~2 dk
bash
$ls /etc /yanlis_klasor > basarili.txt 2> hatalar.txt
$cat basarili.txt
/etc:
passwd shadow hosts ...
$cat hatalar.txt
ls: cannot access '/yanlis_klasor': No such file or directory
Başarılı sonuçlar basarili.txt'e, hatalar hatalar.txt'e gitti. İkisi tamamen ayrı tutuldu.
Zor

Gerçek dünya log sistemi — stdout ve stderr ayrı

~3 dk
bash
$find /root -name "*.conf" > bulunanlar.txt 2> erisim_hatalari.txt
$echo "Bulunan: $(wc -l < bulunanlar.txt) dosya"
Bulunan: 14 dosya
$echo "Hata: $(wc -l < erisim_hatalari.txt) erişim reddi"
Hata: 3 erişim reddi
$# Hataları tamamen yok say:
$find /root -name "*.conf" > bulunanlar.txt 2>/dev/null
2>/dev/null → Linux'ta /dev/null "kara delik" dosyasıdır, oraya yazılan her şey kaybolur. Hataları görmezden gelmek için kullanılır.
🔧
Gerçek Hayat: CI/CD Pipeline & Cron Job'larGitHub Actions, Jenkins gibi sistemler başarılı çıktı ile hata loglarını kesinlikle ayırır. Cron job'lar genellikle 2>/dev/null ile gereksiz uyarıları filtreler.

05 Hepsini Birleştiren Senaryo

Sistem yöneticisi olarak gece otomatik çalışan bir yedekleme scripti yazıyorsunuz.

yedekleme.sh
#!/bin/bash
# ─────────────────────────────────────────────────────
# Otomatik Yedekleme Scripti
# stdin + stdout + stderr hepsini kullanır
# ─────────────────────────────────────────────────────
TARIH=$(date +%Y-%m-%d)
LOG="yedek_${TARIH}.log"
# stdout → log dosyasına ekle (>>)
echo "=== Yedekleme başladı: $TARIH ===" >> "$LOG"
# stdin → yedeklenecek dizinleri dosyadan oku (<)
while read dizin; do
    echo "Yedekleniyor: $dizin" >> "$LOG"
    cp -r "$dizin" /backup/ >> "$LOG" 2>> "hatalar_${TARIH}.log"
done < yedeklenecek_dizinler.txt
# Özet: kaç hata oluştu? (stdin < + stdout >>)
HATA=$(wc -l < "hatalar_${TARIH}.log")
echo "Tamamlandı. Hata sayısı: $HATA" >> "$LOG"
STDIN kullanımı
done < yedeklenecek_dizinler.txt
STDOUT kullanımı
echo "..." >> "$LOG"
STDERR kullanımı
2>> "hatalar_${TARIH}.log"
🏢
Bu script nerede çalışır?Crontab'a eklenerek her gece 02:00'de otomatik çalıştırılır: 0 2 * * * /home/user/yedekleme.sh
